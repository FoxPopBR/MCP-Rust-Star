# Ubiquitous Language & Context: Rust Star Ecosystem

This document defines the shared vocabulary and domain boundaries for the projects managed by the MCP Rust Star Knowledge Server.

## 1. Core Projects

### Rust Star
- **Description**: The primary project, likely a game or application using Rust.
- **Context**: Lore, game mechanics, Rust code standards, and engine documentation.
- **Path**: `C:\Phantasy\Rust Star`

### FoxOT (Open Tibia)
- **Description**: A project related to Open Tibia servers.
- **Context**: Server-side scripting (Lua/C++), game world rules, map lore, and client-server protocol.
- **Path**: `C:\Phantasy\FoxOT`

### FoxClient
- **Description**: The custom client interface for FoxOT.
- **Context**: UI/UX logic, rendering engine (DirectX/OpenGL/Vulkan), and asset management.
- **Path**: `C:\Phantasy\FoxClient`

## 2. Technical Vocabulary

- **Embedding**: The numerical representation of text data, generated by `qwen3-embedding:4b`.
- **RAG (Retrieval-Augmented Generation)**: The process of fetching relevant project context to inform the LLM's response.
- **Project Isolation**: The strict filtering of embeddings by `project_id` to ensure information does not leak between different study subjects.
- **MCP (Model Context Protocol)**: The standard protocol used to expose these knowledge tools to AI agents.
- **Evidence-First**: The development philosophy of always checking logs and reproducing bugs before attempting fixes.
