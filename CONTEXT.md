## 2. Technical Vocabulary

- **Embedding**: The numerical representation of text data, generated by `qwen3-embedding` models.
- **RAG (Retrieval-Augmented Generation)**: The process of fetching relevant project context to inform the LLM's response.
- **Project Isolation**: The strict filtering of embeddings by `project_id` to ensure information does not leak between different study subjects.
- **PostgreSQL + pgvector**: The industrial persistence layer used for vector storage, providing ACID compliance and isolation through project-specific tables.
- **Subdivisão Recursiva (Recursive Chunking)**: Strategy to split large files into smaller fragments automatically if the initial embedding fails due to size/memory constraints.
- **Ejeção de VRAM (VRAM Unloading)**: Proactive unloading of Ollama models between heavy indexing tasks to prevent GPU memory saturation.
- **Shared State (Dashboard)**: Real-time telemetry bridge between the MCP server and the monitoring UI using a shared JSON state and non-blocking log reading.
- **Engineering Whitelist**: A curated list of relevant file extensions (`.otmod`, `.otui`, `.lua`, `.rs`, `.cpp`, `.frag`, `.vert`, etc.) used to filter out noise from large binary asset folders.
- **MCP (Model Context Protocol)**: The standard protocol used to expose these knowledge tools to AI agents.
- **Evidence-First**: The development philosophy of always checking logs and reproducing bugs before attempting fixes.
